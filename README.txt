    Данная библиотека реализует основные объекты и взаимодействия между ними необходимые для организации работы
торгового бота и является частью микросервисной архитектуры.

    Особенности проекта:
    1. Наличие сессии пользователей. Невостребованные данные пользователей не хранятся в оперативной памяти сервера,
а могут быть автоматически удалены после истечения срока их хранения в оперативной памяти. При этом гарантируется их
сохранность и восстановление при последующих обращениях пользователей.
    2. Удобный интерфейс для просмотра каталога товаров, корзины и заказов пользователей.

    Для запуска проекта перейдите в каталог с докерфайлом проекта и выполните команды:
    docker build -t bot_shop .
    docker run bot_shop
